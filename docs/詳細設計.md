## 2. システム構成

```
Flutter App (Web / Android / iOS)
       │
       ├── Firebase Authentication  (認証)
       ├── Cloud Firestore          (データストア)
       ├── Firebase Storage         (画像ストレージ)
       └── Firebase Messaging       (プッシュ通知 / PS_07)
```

**地図**: OpenStreetMap (flutter_map) — Google Maps API は不使用  
**Firebase プロジェクト**: `flutter-afterglow`

## 5. 画面設計

### 5.1 LoginPage（FR_01）

**入力項目**: メールアドレス、パスワード  
**アクション**: ログイン（Firebase Auth `signInWithEmailAndPassword`）、ユーザ登録ページへのリンク  
**バリデーション**: メール形式チェック、パスワード 8 文字以上

### 5.2 RegisterPage（FR_08）

**入力項目**: ユーザ名、メールアドレス、パスワード  
**アクション**: 新規登録（`createUserWithEmailAndPassword` → `/users/{uid}` に初期ドキュメント作成）

### 5.3 FeedPage（FR_02）

**表示**: 最新投稿順 `ListView`（`getPosts()` の Stream を購読）  
**各カード**: サムネイル（先頭画像）、投稿者名、caption、タグ、いいね数、コメント数  
**ページネーション**: `Firestore.limit(20)` + スクロール末端での追加取得

### 5.4 MapScreen（FR_03）— 既存

**拡張内容**:  
- 地図タップ時の `PostAddDialog` → 投稿完了後は Dialog ではなく `PostDetailPage` に遷移  
- GPS ボタン追加（`geolocator` を有効化し現在地へ移動）  
- マーカータップ → Dialog から `PostDetailPage` への Navigator.push に変更（要件定義の画面遷移図に準拠）

### 5.5 PostAddPage / PostAddDialog（FR_07）— 既存拡張

**追加入力項目**:
- 場所名（テキストフィールド。GPS 取得ボタンで自動入力補助）
- タグ（`#タグ名` で複数入力可。ChipInput 実装）

**バリデーション**:
- 画像: 1〜10枚（NFR_02）
- ファイル拡張子: JPEG / PNG / BMP / HEIC（NFR_02）
- 画像サイズ: 1枚 10MB、全体 100MB（NFR_02）

**画像圧縮**（NFR_02）: アップロード前に `flutter_image_compress` で最大短辺 2048px・品質 85% にリサイズ。元の解像度超は等倍のまま。

### 5.6 PostDetailPage（FR_04）— 現状の PostCardView を拡張

**表示内容**:
- 画像ギャラリー（PageView、現状実装済み）
- 投稿者プロフィール（アイコン + ユーザ名 → ProfilePage へ遷移）
- キャプション（現状実装済み）

- タグ一覧（ChipRow）
- リアクションバー（いいね数 + ボタン）
- コメントセクション（ListView + 投稿フォーム）
- 自投稿の場合のみ編集・削除メニュー（PS_08）

### 5.7 EventPage（FR_05）

**表示**: イベント一覧（開催日昇順）  
**詳細**: イベント名、日時、場所（地図ミニプレビュー）、参加者一覧、コメント

## 7. セキュリティ設計（NFR_01）

### 7.1 Firebase Authentication

- メール/パスワード認証（`firebase_auth` パッケージ）
- パスワードはクライアントで平文送信 → Firebase Auth が bcrypt でハッシュ化して保存

### 7.2 Firestore セキュリティルール（未設定、要実装）

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // ユーザー: 本人のみ書き込み
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;
    }

    // 投稿: 読み取りは認証済み、書き込みは本人のみ
    match /posts/{postId} {
      allow read: if request.auth != null;
      allow create: if request.auth != null;
      allow update, delete: if request.auth.uid == resource.data.userId
                            || isAdmin();

      // コメント: 読み取りは認証済み
      match /comments/{commentId} {
        allow read: if request.auth != null;
        allow create: if request.auth != null;
        allow update, delete: if request.auth.uid == resource.data.userId;
      }

      // リアクション: 1ユーザー1ドキュメント
      match /reactions/{userId} {
        allow read: if request.auth != null;
        allow write: if request.auth.uid == userId;
      }
    }

    // 通知: 受信者のみ読み取り
    match /notifications/{notificationId} {
      allow read: if request.auth.uid == resource.data.recipientId;
      allow write: if false; // Cloud Functions のみ書き込み
    }

    function isAdmin() {
      return get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
  }
}
```

### 7.3 Firebase Storage セキュリティルール

```
rules_version = '2';
service firebase.storage {
  match /b/{bucket}/o {
    match /posts/{userId}/{allPaths=**} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId
                   && request.resource.size < 10 * 1024 * 1024  // 10MB
                   && request.resource.contentType.matches('image/.*');
    }
    match /profiles/{userId}/{allPaths=**} {
      allow read: if request.auth != null;
      allow write: if request.auth.uid == userId;
    }
  }
}
```

## 8. パフォーマンス設計（NFR_02）

### 8.1 画像アップロード圧縮

アップロード前に `flutter_image_compress` パッケージで処理する。

| 条件 | 処理 |
|---|---|
| 短辺 > 2048px | 短辺 2048px にリサイズ（アスペクト比維持）|
| 短辺 ≤ 2048px | リサイズなし |
| 品質 | JPEG 品質 85% |
| ファイルサイズ | 圧縮後でも 10MB 超の場合はアップロード拒否 |

### 8.2 フィードページ

- Firestore クエリに `limit(20)` を設定（NFR_02 の3秒以内目標）
- スクロール末端で `startAfterDocument` による追加取得（無限スクロール）
- 画像は `CachedNetworkImage` でディスクキャッシュ

### 8.3 地図ページ

- タイルは `flutter_map` の標準キャッシュを使用（NFR_02 遅延読み込み）
- 画像プリロードは実装済み（MapScreen の `precacheImage` ループ）
